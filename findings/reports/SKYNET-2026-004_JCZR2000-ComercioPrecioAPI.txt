WHAT I FOUND:
Your scraper.py script uses a GITHUB_TOKEN environment variable to push
exchange rate data to a GitHub repository. While the token value is not
hardcoded (good!), the script structure reveals important security concerns:

1. The token needs WRITE access to push data to your repository
2. If this token is exposed, an attacker can modify your code
3. If the token has access to other repositories, they are also at risk
4. The script scrapes BCV and Binance P2P — sensitive economic data

File: scraper.py

  REPO_NAME = os.getenv("REPO_NAME")

HOW TO FIX:
1. Use GitHub Fine-grained Personal Access Tokens (not classic tokens)
2. Restrict the token to ONLY the specific repository
3. Set the minimum required permissions (Contents: Read and Write)
4. Set an expiration date (90 days max)
5. Monitor token usage in GitHub Settings -> Security log
6. Consider using GitHub Actions with repository secrets instead
