  FINDING: Hardcoded JWT Secret Key Pattern
  Severity: HIGH
  CWE: CWE-798

WHAT I FOUND:
Your repository implements JWT authentication using a secret key stored in
settings.secret_key. This key is used to sign and verify all authentication
tokens in your API.

The concern: if this key is predictable, default, or has ever been committed
to the repository (even in a previous commit), an attacker can forge valid
authentication tokens and impersonate any user.

AFFECTED FILES:
- bcv_api/security.py (lines 50, 83, 105)
- bcv_api/config.py

AFFECTED ENDPOINTS:
- /token [POST] — Any attacker with the key can generate valid tokens
- /users/me [GET] — User data extraction
- / [POST] — Data manipulation

HOW TO FIX:
1. Generate a strong random secret: python -c "import secrets; print(secrets.token_hex(32))"
2. Store it ONLY in environment variables, never in code
3. Add startup validation:
   if not settings.secret_key or settings.secret_key == "change-me":
       raise RuntimeError("JWT_SECRET must be set in environment")
4. Rotate all existing tokens
5. Check git history: git log --all --diff-filter=M -- bcv_api/config.py
